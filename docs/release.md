# 发布说明

本文件同时承担两类职责：

- 作为当前仓库的版本记录
- 作为未来发版时的检查清单

## 当前状态

### 文档基线发布日期

- 日期：`2026-03-06`
- 版本标识：`docs-refresh-2026-03-06`

### 当前可对外说明的能力

- 登录与角色区分（Admin / Member）
- 知识库与文档管理
- 文档上传、异步入库、阶段追踪
- 文档在线预览与小型 TXT 在线编辑
- 会话管理、普通问答、流式问答、消息反馈
- 统一日志查看与导出

### 当前不应对外承诺的能力

- 企业级多租户
- 完整 SSO / OAuth / LDAP
- 高可用集群部署
- 生产级备份恢复演练
- 完整安全基线与审计合规

## 建议的版本策略

建议使用下列版本语义：

- `major`：接口或部署方式发生不兼容变化
- `minor`：新增功能或新增文档模块
- `patch`：缺陷修复、文档修正、脚本小幅改进

## 发版前检查清单

### 代码与配置

- `.env.example` 与实际配置项一致
- 默认密码和敏感值没有被误写入公开文档
- `docker compose config --quiet` 通过

### 测试与校验

- `python scripts/check_encoding.py`
- `cd services/go-api && go test ./...`
- `cd services/py-rag-service && python -m pytest -q`
- `cd services/py-worker && python -m pytest -q`
- `cd apps/web && npm run build`

### 文档

- 根 README 与代码行为一致
- `docs/README.md` 中的链接有效
- `docs/API_SPECIFICATION.md` 与当前路由一致
- `docs/runbook.md` 与当前日志脚本一致
- `docs/openapi.yaml` 至少同步到当前公开接口边界

### 发布素材

- 仓库标题、副标题、关键词已准备
- 演示脚本可直接跑通
- 如准备公开发布，已补充许可证文件

## 推荐的变更记录格式

每次发布建议记录四项：

- `What`：做了什么
- `Why`：为什么做
- `How to verify`：如何验证
- `Risk`：风险与回滚方式
