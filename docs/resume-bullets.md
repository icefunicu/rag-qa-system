## 50 字以内短介绍

- 一个可运行的私有知识库问答系统，重点解决文档入库黑盒、问答缺依据和排障难的问题。

## 120 字左右介绍

- RAG-QA System 是一个基于 Go、Python 与 Vue 的私有知识库问答系统。它不仅支持文档上传与问答，还提供异步入库状态追踪、文档预览、会话持久化和统一日志排障能力，适合作为团队内部知识库平台的起点工程或演示型全栈 RAG 项目。

## 平台长描述

### GitHub / Gitee 仓库介绍

> RAG-QA System 是一个工程化的私有知识问答系统。项目采用 Go API Gateway、Python RAG Service、Python Worker 和 Vue Web Console 组成完整链路，覆盖知识库管理、文档上传、异步入库、在线预览、会话问答、引用溯源和日志排障。相比只有聊天接口的 RAG Demo，这个仓库更强调可见的状态机、明确的服务边界和可执行的运维文档。

### 博客或项目主页介绍

> 这个项目关注的不是“再做一个聊天框”，而是把 RAG 真正做成可交付的系统。上传文档后，用户可以直接看到当前任务是否停留在解析、切块、向量化还是校验阶段；管理员可以在页面里拿到任务 ID 和排障命令；聊天链路则通过 Go 网关统一处理认证、会话和转发。它适合用来展示 RAG 在工程层面的完整落地方式。

## 项目亮点要点

- 预签名上传 + 异步 Worker 入库，避免大文件穿透 API 网关
- 文档入库阶段显式化，支持 Timeline 追踪
- 文本预览支持编码探测，降低中文 TXT 乱码概率
- 聊天会话、消息和反馈持久化
- 提供统一日志脚本和导出脚本，排障路径清晰

## 简历表达建议

### 一行版

- 设计并实现基于 Go、Python、Vue 的私有知识库问答系统，覆盖文档异步入库、状态追踪、在线预览、会话持久化与日志排障。

### 两行版

- 负责构建工程化 RAG 系统，采用 Go API Gateway、Python 检索服务与异步 Worker 分层架构，支撑文档上传、向量入库、问答与反馈闭环。  
- 重点完善文档入库可观测性与运维可追踪性，支持任务时间线、任务 ID 定位和统一日志导出，降低上传与索引问题排查成本。
