# 演示脚本

本文档用于 5 分钟以内的产品演示、面试介绍或仓库讲解。目标不是讲所有细节，而是让观众快速理解这不是一个只有聊天框的 RAG Demo。

## 演示目标

在有限时间里让观众看到四件事：

1. 这是一个完整系统，而不是单接口调用
2. 文档上传与入库过程可见
3. 问答结果有作用域和引用概念
4. 出问题时有日志和时间线，不是黑盒

## 演示前准备

- 本地环境已通过 `make up` 启动
- 使用 Admin 账号登录
- 准备一个小型 TXT 和一个较大的 TXT 或 PDF
- 打开 `.\logs.bat -f -s go-api py-worker` 备用

## 演示顺序

### 第 1 段：系统入口（30 秒）

页面：登录页 -> 主界面

话术：

> 这套系统不是只做“提问”。它把知识库管理、文档处理、问答和排障入口放在同一个控制台里。普通成员可以直接聊天，管理员还能管理知识库和查看文档入库状态。

### 第 2 段：上传文档与可观测性（90 秒）

页面：知识库管理 -> 某个知识库详情 -> 上传文档

动作：

1. 上传一个文档
2. 展示页面中的阶段、任务 ID、文档 ID
3. 打开“详情”，展示 Ingest Timeline

话术：

> 这里最重要的不是“可以上传”，而是“上传之后发生了什么是可见的”。系统会告诉你当前卡在下载、解析、切块还是写入向量，而不是只给一个模糊的处理中状态。

### 第 3 段：在线预览与大文件策略（45 秒）

页面：文档列表 -> 在线查看

动作：

1. 打开一个小 TXT，展示直接文本预览
2. 打开一个大 TXT，展示只读预览、编码提示或部分预览提示

话术：

> 小文件可以直接编辑，大文件则会切到只读预览或部分预览模式，避免浏览器卡死。系统还会展示检测到的编码，减少中文文本乱码问题。

### 第 4 段：基于知识库发起问答（90 秒）

页面：聊天页

动作：

1. 新建会话
2. 指定知识库范围提问
3. 展示返回结果和引用信息

话术：

> 这里的重点不是模型答了什么，而是答案来自哪个知识范围。作用域和引用让问答结果更容易被验证和追责。

### 第 5 段：演示排障能力（45 秒）

页面：上传组件 + 本地日志终端

动作：

1. 指向上传组件中的日志命令提示
2. 展示日志窗口里可直接按 `job_id` 或 `document_id` 过滤

话术：

> 真正的工程差异体现在故障发生时。这里不仅能看到任务 ID 和阶段，还能直接给出日志命令，开发或运维接手时不需要再从零猜。

## 版本化演示路线

### 面向技术面试

强调：

- 上传链路是异步化的
- 状态机和时间线是可见的
- 文本预览做了编码探测
- Go / Python / Vue 的服务边界清晰

### 面向业务或产品演示

强调：

- 文档从上传到可问答是完整流程
- 管理员与普通成员的界面边界不同
- 大文件和错误场景有兜底策略

### 面向仓库发布页面

强调：

- 该项目包含可运行的全链路系统
- 不只是检索问答，还包括文档管理、入库可观测性和排障工具

## 一句话总结模板

> 这是一个工程化的 RAG 知识库系统，重点价值不只是“能回答”，而是“上传、索引、预览、问答和排障都可见、可控、可验证”。
