# 安全策略

## 支持中的版本

当前仓库只维护默认分支。

| 版本 | 是否支持 |
| --- | --- |
| `main` / 默认分支 | 是 |
| 历史快照与分叉仓库 | 否 |

## 漏洞上报方式

请不要在公开 Issue 中直接披露敏感漏洞细节。

建议的上报顺序：

1. 如果仓库已启用 GitHub 私密漏洞上报，优先使用该渠道
2. 如果没有私密上报入口，请通过维护者的私下渠道联系
3. 如果两者都不可用，请先提交一个不含利用细节的公开 Issue，并说明需要私下跟进

上报时建议包含：

- 受影响的组件或路径
- 影响范围概述
- 复现步骤
- 必要时提供最小 PoC
- 如果已有思路，可附上修复建议

## 报告中不要包含的内容

- 真实密钥、生产环境凭据或私有文档
- `.env` 文件
- 在修复可用之前，完整公开利用链

## 响应目标

仓库当前采用 best-effort 方式处理安全问题，目标流程如下：

- 确认收到报告
- 评估影响范围
- 复现问题
- 准备修复
- 如有必要，补充修复说明或缓解建议

当前仓库不承诺固定 SLA。
